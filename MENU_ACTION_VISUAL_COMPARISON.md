## Code Comparison: What Works vs What Doesn't

### ‚ùå DOESN'T WORK: Imperative Menu Action

```typescript
// src/main.tsx
import { Devvit } from '@devvit/public-api';

Devvit.configure({
  redis: true,
  http: true,
  redditAPI: true,
});

// ‚ùå This does NOT show in UI
Devvit.addMenuItem({
  label: 'Create a new post',
  location: 'subreddit',
  forUserType: 'moderator',
  onPress: async (event, context) => {
    try {
      const subreddit = await context.reddit.getCurrentSubreddit();
      await context.reddit.submitPost({
        title: 'New Mystery Case',
        subredditName: subreddit.name,
        preview: (
          <vstack>
            <text>Loading mystery...</text>
          </vstack>
        ),
      });

      context.ui.showToast('Post created!');
    } catch (error) {
      context.ui.showToast(`Error: ${error.message}`);
    }
  },
});
```

**Result**:
- ‚ùå Menu action does NOT appear in UI
- ‚ùå User can't click it
- ‚ùå Handler never executes
- ‚úÖ Code compiles without errors (misleading!)
- ‚úÖ Appears in built bundle (but doesn't register)

**Why it fails**:
1. Menu UI rendered before main.tsx executes
2. Custom Post context can't control subreddit-level menu
3. Platform doesn't route imperative menu actions from main.tsx

### ‚úÖ WORKS: Declarative Menu Action + Scheduler Bridge

#### Step 1: Declare menu action in devvit.json

```json
{
  "menu": {
    "items": [{
      "label": "Create a new post",
      "description": "Generate a new murder mystery case",
      "location": "subreddit",
      "forUserType": "moderator",
      "endpoint": "/internal/menu/post-create"
    }]
  }
}
```

**Result**:
- ‚úÖ Menu action APPEARS in UI
- ‚úÖ User can click it
- ‚úÖ Routes to Express endpoint

#### Step 2: Handle menu action in server context

```typescript
// src/server/index.ts
import { reddit, scheduler, redis } from '@devvit/web/server';
import express from 'express';

const router = express.Router();

router.post('/internal/menu/post-create', async (req, res) => {
  try {
    console.log('üìç Menu action triggered');

    // ‚ùå Can't use reddit.submitPost() here - no Reddit API access
    // ‚úÖ Instead, trigger scheduler job that runs in main context

    await scheduler.runJob({
      name: 'daily-case-generation',
      runAt: new Date(Date.now() + 1000), // Run in 1 second
      data: {
        source: 'menu-action',
        triggeredAt: new Date().toISOString(),
      }
    });

    console.log('‚úÖ Scheduler job triggered');

    // Return success to menu action (shows toast)
    res.send({
      success: true,
      message: 'Case generation started! Check back in 30-60 seconds.',
```

## Summary: The Mental Model

**Wrong Mental Model**:
```
My app = One thing with configuration
  ‚îî‚îÄ All features share configuration
       ‚úÖ Works in Standard Devvit
       ‚ùå Doesn't work in Web Framework
```

**Correct Mental Model**:
```
My app = Two separate contexts
  ‚îú‚îÄ Main context (dist/main.js)
  ‚îÇ    ‚úÖ Has configuration
  ‚îÇ    ‚úÖ Has Reddit API
  ‚îÇ    ‚úÖ Renders Custom Posts
  ‚îÇ    ‚úÖ Runs scheduler jobs
  ‚îÇ
  ‚îî‚îÄ Server context (dist/server/index.cjs)
       ‚ùå Has no configuration
       ‚ùå No Reddit API access
       ‚úÖ Handles menu actions
       ‚úÖ Uses scheduler bridge to reach main context
```

**The Bridge Pattern**:
```
User clicks menu
  ‚Üí Server context (handles click)
    ‚Üí Scheduler job (bridges contexts)
      ‚Üí Main context (has Reddit API)
        ‚Üí Creates post
          ‚Üí User sees result
```
